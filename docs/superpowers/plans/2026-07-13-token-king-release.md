# Token King Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** 建立唯一、可重复、可验证的 Token King universal 签名公证发布链路，并让本地安装、GitHub Release 和 Sparkle 更新使用同一版本事实源。

**Architecture:** Xcode target 的 `MARKETING_VERSION`/`CURRENT_PROJECT_VERSION` 是版本 SSOT；release workflow 只消费精确 tag，并通过脚本完成 fail-fast 断言、产物校验和发布后回读验证。

**Tech Stack:** Xcode/xcodebuild, GitHub Actions, codesign, notarytool, hdiutil/create-dmg, Sparkle 2, shell tests/action-validator.
