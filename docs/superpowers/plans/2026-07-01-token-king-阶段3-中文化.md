# Token King — 阶段 3（中文化）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 Token King 的菜单栏所有 user-facing 英文文案改成中文（直接中文字面量方案），品牌专有名词、内部数据标识符、日志保持不动。

**Architecture:** 直接中文字面量。逐文件把 user-facing 英文字符串（菜单标题、菜单正文标签、alert/按钮文案、枚举显示名、含插值串的英文前缀词）替换为中文。**否决了设计文档原定的 String Catalog（.xcstrings）方案**——理由见下方 ADR。验证靠 build + grep 残留扫描 + 人工看菜单，不套 TDD（纯 UI 文案无法有意义地单测）。

**Tech Stack:** Swift 5 / Xcode / xcodebuild。工程在 `~/projects/usage-deck/CopilotMonitor/`，scheme = `CopilotMonitor`，macOS 13+。

## ADR：为什么否决 String Catalog，改用直接中文字面量

设计文档（`docs/superpowers/specs/2026-06-30-token-king-fork设计.md` 阶段3）原定"接入 String Catalog（.xcstrings）"。摸底后否决，改直接中文字面量，理由：
1. **手动 pbxproj**：本工程 pbxproj 全手动管理（阶段0/1/2 已验证），.xcstrings 的 Xcode 原生集成（自动提取字符串、build 阶段处理）在命令行 + 手动 pbxproj 下难可靠落地。
2. **海量插值**：13 个插值标题 + 大量 `createDisabledLabelView` 正文标签用 Swift 字符串插值（`"Credits: \(...)"`），NSLocalizedString/String Catalog 对插值支持别扭，要么重构代码，要么效果差。
3. **单语言目标**：个人 fork，目标是"菜单全中文"，不需要运行时中英切换。String Catalog 的主要好处（多语言分离、运行时切换）用不上，却要付集成成本。

代价（已接受）：放弃运行时语言切换能力。若将来要做多语言，届时另写 ADR 重新引入 String Catalog。

## 【全局铁律】绝对不能翻译的东西（每个 task 都必须遵守）

以下是**内部标识/品牌/日志**，翻译会破坏逻辑或不符合规范，任何 task 都**禁止**改动：

1. **数据标识符字面量**：`authSource: "OpenCode"`、`identifier == "..."`、`rawValue`、`case xxx = "..."` 的 rawValue。这些是 provider 匹配逻辑的键，翻译会导致匹配失败。
   - 判断法：字符串在被**比较**（`==`）、**赋值给 authSource/identifier/rawValue**、或**作为 dict key** 时，是标识符，不翻。
2. **provider displayName 返回值**（`ProviderProtocol.swift:35-79`）：`"GitHub Copilot"` / `"Claude"` / `"ChatGPT"` / `"Tavily"` / `"Brave Search"` 等全是**品牌专有名词**，不翻。
3. **日志文案**：`logger.info(...)` / `logger.error(...)` / `debugLog(...)` / `Logger(subsystem:category:)` 里的英文，全部保持英文，不翻（内部日志，规范要求英文便于检索）。
4. **subsystem/category 字符串**：`"com.opencodeproviders"` 等，不翻。
5. **SF Symbol 名**：`"key"` / `"crown"` / `"person.circle"` / `"arrow.triangle.branch"` / `"number.circle"` 等是系统图标名，不翻。
6. **时间单位缩写**：`RefreshInterval` 的 `"1m"` / `"3m"` / `"1h"` 保持（通用缩写，翻成"1分钟"反而累赘）；但**带英文单词**的翻，如 `"7 days"` → `"7 天"`。
7. **品牌名 "OpenCode Bar" / "OpenCode"**：**本 task 不动**（阶段0 已把 bundle 层改 Token King；user-facing 短标题的品牌改名单独在 Task 3.5 处理，且需区分 user-facing vs 数据标识）。

**每个 task 改完后必须自查**：`git diff` 里有没有误改上述任何一类。误改了立即还原。
