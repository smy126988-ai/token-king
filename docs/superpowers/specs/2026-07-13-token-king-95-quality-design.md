# Token King 95 分产品化总设计

> 日期：2026-07-13
> 基线：`main@e8a2f31`
> 目标：把当前自用开发版收敛为数据可信、运行稳定、易理解、可验证、可安装更新的 macOS 产品。

## 1. 当前判断

当前综合评分为 52/100。现有多 Provider、额度监控、人民币、Token 归集和菜单栏体验构成了有效产品基础，但以下问题阻止发布：

- 月度 Token 少算约 84%；
- 新增模型价格与真实 App 计算链路脱节；
- API 快照不会更新，部分 Provider 归属错误；
- 默认测试可能访问真实 Provider，且 44 个测试方法未加入 target；
- 生产路径持续写入包含账号和本地路径的无限日志；
- README、版本、Release workflow、应用产物和更新源相互不一致。

## 2. 95 分硬门槛

95 分不是按功能数量累加，而是同时通过以下 Gate：

| Gate | 必须满足 |
|---|---|
| 数据可信 | 脱敏真实 fixture 与 CTCA 对账误差不超过 1%；未知价格不按零静默计入 |
| 聚合完整 | 当月 `day_aggregates` 与 `token_events` 五个 token 字段逐项相等 |
| 安全性能 | 默认零凭证片段日志；空闲 CPU <2%；无变化刷新 p95 <1 秒；24 小时日志增长 <5MB |
| 测试质量 | 默认测试零真实网络；磁盘测试文件全部进入 target；关键菜单有确定性 UI 验收 |
| 产品体验 | 首屏 10 秒内回答本月 Token、实际支出、API 折算、订阅成本；统计区具备五态 |
| 发布可用 | 版本、tag、DMG、appcast、运行 commit 一致；universal、签名、公证、干净安装通过 |
| 运行稳定 | 当前 HEAD 冷启动、登录启动、休眠唤醒和可用显示器矩阵无重复/消失/不可点击图标 |

任一 P0 或 P1 未关闭时，最高不得评为 95。

## 3. 子项目与顺序

| 顺序 | 子项目 | 独立产物 |
|---:|---|---|
| 1 | 数据与计价正确性 | typed pricing contract、真实 fixture、覆盖率可见的成本结果 |
| 2 | 聚合与刷新管线 | 历史日回填、快照语义、增量 checkpoint、单次 UI 刷新 |
| 3 | 安全、测试与运行稳定 | 离线测试、日志收口、build hash、状态栏 bridge 隔离 |
| 4 | 产品体验与文档 | 统计五态、Provider 显隐、中文收口、README/backlog/截图 |
| 5 | 发布与更新 | 唯一版本源、Token King workflow、签名公证、Sparkle、安装验收 |

每个子项目必须先完成自己的测试和 review，才能进入下一个。发布 Gate 在前四项通过后执行。

## 4. 公共设计原则

- Provider 是计费主体，Tool/Source 是数据来源；二者不可混用。
- 金额必须携带币种和来源，禁止依赖无单位 `Double` 或硬编码汇率。
- 统计必须显式表达完整、部分、未知、失败和缓存状态。
- 数据正确性测试使用脱敏 fixture 和独立 oracle，不用同一实现生成期望值。
- 默认测试、构建和截图不得依赖用户凭证或外部网络。
- 保持现有不可变菜单组标题，不用随机空格对齐，不以文本颜色承担语义。
- 生产日志不记录 token、cookie、key 片段、邮箱或完整本地路径。

## 5. 验证与评分

每个 Gate 完成后更新一份评分矩阵：

| 维度 | 权重 | 95 分要求 |
|---|---:|---|
| 数据正确性 | 30 | 29+ |
| 稳定性与性能 | 20 | 19+ |
| 产品体验 | 15 | 14+ |
| 测试与可维护性 | 15 | 14+ |
| 发布与文档 | 20 | 19+ |

最终必须以 fresh build、fresh test、真实 fixture、运行日志、渲染截图和发布产物为证据，不能只靠代码 diff 或单元测试数量。

## 6. 范围外

- 不在本轮开发桌面 Widget；先把菜单栏产品做可信。
- 不为没有官方价格的模型编造代表价；显示“未计价”。
- 不为追求文件行数而做与当前 Gate 无关的全仓重写。
- 不改变 `AGENTS-design-decisions.md` 中标记为 immutable 的菜单业务规则。
